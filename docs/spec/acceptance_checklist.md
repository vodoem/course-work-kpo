# Чек-лист приёмки (быстрый)

## Архитектура и структура
- [ ] MVC соблюдён; View не содержит логики игры.
- [ ] Общая логика вынесена в отдельные библиотеки.
- [ ] Консоль и Avalonia используют одни и те же Model/Controller.
- [ ] Взаимодействие — через собственные события (а не прямые зависимости UI -> Model).
- [ ] Использовано минимум 2 паттерна сверх MVC (например Observer + Factory Method).

## Потоки и синхронизация
- [ ] Есть минимум 2 потока (например: game loop + UI/input).
- [ ] Синхронизация доступа к состоянию (lock/immutable snapshots/каналы и т.п.).
- [ ] Таймеры не используются.

## Игровая логика
- [ ] dt=1/60, обновление детерминированное.
- [ ] Скорости растут по формуле v = v_base * (1 + 0.015*(level-1)).
- [ ] Коллизии AABB (+8px soft margin).
- [ ] Пауза P/Esc, возобновление с 3…1.

## Рекорды
- [ ] `records.json` создаётся/читается автоматически.
- [ ] Формат соответствует `docs/spec/records_format.md`.

## Тесты
- [ ] Выбран один класс и покрыт ≥20 unit-тестами.
- [ ] Тесты детерминированы (рандом через IRandomProvider/seed).
