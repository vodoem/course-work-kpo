# Codex Execution Plans (ExecPlans)

Этот файл — шаблон для больших задач, когда «просто промпта» недостаточно.
ExecPlan — самодостаточная спецификация, по которой агент может реализовать фичу от начала до конца.

## Когда использовать
- новая крупная подсистема (например, полностью новый UI-экран, новый формат хранения)
- сложный рефакторинг архитектуры
- задача, где есть риск нарушить ограничения (потоки, «без таймеров», разделение по проектам)

## Шаблон ExecPlan
Скопируй в `plans/XXXX-<slug>.execplan.md` и заполни:

1. **Goal**
2. **Non-goals**
3. **Source of truth** (какие docs/файлы обязательны)
4. **Constraints** (без таймеров, MVC, 2 UI, net8.0, стиль, тесты)
5. **Design**
   - структуры данных
   - границы проектов
   - события
   - потоки и синхронизация
6. **Milestones**
   - M1: ...
   - M2: ...
7. **Validation**
   - команды сборки/тестов
   - ручной чек-лист
8. **Open questions / Decisions log**

Примечание: ExecPlan должен быть самодостаточным — чтобы новичок мог выполнить работу, имея только репозиторий и этот файл.
